## What I do

- Start a local Cloudflare Workers dev server with hot reload
- Build Worker bundles for production deployment
- Deploy Workers to Cloudflare infrastructure
- Manage environment secrets and configuration
- Generate TypeScript bindings for Cloudflare APIs

## When to use me

Use this skill when:

- Running the Worker locally during development (`bun w dev`)
- Building a production Worker bundle (`bun w build`)
- Deploying to Cloudflare (`wrangler deploy`)
- Adding or updating secrets (`wrangler secret put`)
- Generating types for Cloudflare bindings (`wrangler types`)

## Commands Reference

### Development

```bash
bun w dev                    # Start local dev server (localhost:8787)
bun w dev --local           # Force local-only (no remote binding)
bun w dev --env staging     # Target specific environment
```

### Build & Deploy

```bash
bun w build                 # Build Worker bundle for deployment
wrangler deploy             # Deploy to Cloudflare (after build)
wrangler deploy --env prod  # Deploy to production environment
```

### Types

```bash
bun w types                 # Generate types from wrangler.toml (run after schema changes)
bun run generate-types      # npm-equivalent (this project's alias)
```

### Secrets

```bash
wrangler secret put <NAME>   # Set a secret interactively
wrangler secret list          # List all secrets
wrangler secret delete <NAME> # Remove a secret
```

### Other

```bash
wrangler whoami              # Check Cloudflare auth status
wrangler logs                # tail real-time logs from deployed Worker
wrangler rollback            # Rollback to previous deployment
wrangler dev --inspecta      # Enable remote debugging
```

## wrangler.toml

The `wrangler.toml` configures the Worker:

```toml
name = "tremtec-worker"
main = "src/server/index.ts"
compatibility_date = "2024-01-01"

[site]
bucket = "./dist"

[env.staging]
name = "tremtec-staging"

[env.production]
name = "tremtec"
```

## Cloudflare Bindings

Workers access Cloudflare resources via bindings defined in `wrangler.toml`:

```typescript
// Types are generated by `bun w types`
interface Env {
  DB: D1Database;
  KV: KVNamespace;
  ASSETS: R2Bucket;
  AI: Ai;
}

app.get("/api/data", (c) => {
  const result = await c.env.DB.prepare("SELECT * FROM users").all();
  return c.json(result);
});
```

## Environment Variables

Environment variables (non-secrets) go in `wrangler.toml`:

```toml
[vars]
BASE_URL = "https://tremtec.com"
```

For secrets, use `wrangler secret put` — never commit secrets to `wrangler.toml`.
