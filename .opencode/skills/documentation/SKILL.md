## What I do

- Write and maintain README files with clear setup and usage instructions
- Create API documentation with examples and endpoint descriptions
- Generate changelogs and release notes from commit history
- Write inline code comments and JSDoc/TSDoc annotations
- Create user guides and tutorials for features
- Document architecture decisions and technical specifications
- Maintain consistent documentation style across the project

## When to use me

Use this skill when:

- Creating or updating README files
- Writing API documentation for endpoints
- Generating changelogs for releases
- Adding code comments for complex logic
- Creating user-facing documentation
- Documenting architectural decisions (ADRs)
- Onboarding new team members with project docs

## Best practices

- Start with the why before the what in documentation
- Include practical examples and code snippets
- Keep READMEs scannable with clear headings
- Write for the target audience (users vs developers)
- Update docs when code changes
- Use consistent terminology throughout
- Include troubleshooting sections for common issues
